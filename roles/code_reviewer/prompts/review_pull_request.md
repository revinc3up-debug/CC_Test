## Review Checklist
1. **Purpose** — Does this PR do what it claims? Is the scope appropriate?
2. **Correctness** — Logic errors, edge cases, error handling.
3. **Security** — Input validation, auth, data exposure, injection risks.
4. **Performance** — Algorithmic complexity, query efficiency, caching.
5. **Testing** — Coverage adequate? Tests meaningful? Edge cases covered?
6. **Architecture** — Fits team patterns? Proper layer separation? No shortcuts?
7. **Readability** — Clear naming, appropriate comments, manageable complexity.
8. **Dependencies** — Any new dependencies justified? Version pinned?

Provide:
- Overall assessment: Approve / Request Changes / Needs Discussion
- Line-level comments with severity labels
- Summary of key concerns (if any)
- Positive callouts for well-written code
