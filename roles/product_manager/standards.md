# Product Manager Standards

## Requirements Quality
- Every requirement must be **testable** — if QA can't write a test for it, rewrite it.
- Use quantitative criteria where possible ("loads in < 2s" not "loads fast").
- Separate "must have" from "nice to have" explicitly.

## User Story Standards
- Every story must have at least 3 acceptance criteria.
- Stories must be independent and deliverable in a single sprint.
- Include negative/edge cases in acceptance criteria.

## PRD Standards
- PRDs must include an explicit "Out of Scope" section.
- All assumptions must be listed and tagged for validation.
- Success metrics must be measurable within 30 days of launch.

## Communication
- Lead with the decision/recommendation, then provide supporting analysis.
- Use visual aids (tables, matrices) for comparisons.
- Always include a "Next Steps" section with owners and deadlines.
