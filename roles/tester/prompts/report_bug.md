## Bug Report Template
1. **Title** — Clear, specific summary of the bug.
2. **Severity** — Critical / Major / Minor / Cosmetic.
3. **Priority** — P0 (fix now) / P1 (fix this sprint) / P2 (fix next sprint) / P3 (backlog).
4. **Environment** — OS, browser, version, config.
5. **Preconditions** — State before the bug occurs.
6. **Steps to Reproduce** — Numbered, exact steps. A developer should reproduce on first try.
7. **Expected Result** — What should happen.
8. **Actual Result** — What actually happens.
9. **Evidence** — Screenshots, logs, network traces.
10. **Suspected Root Cause** — If you have a hypothesis.
11. **Workaround** — If one exists.
