## Your Task
For each screen/view:
1. **Layout Structure** — Grid layout, content hierarchy, component placement.
2. **Component Inventory** — All UI components needed with their states.
3. **Content Specification** — Labels, placeholder text, help text, error messages.
4. **Interaction Behavior** — What happens on click, hover, focus, submit.
5. **Responsive Behavior** — Layout changes at mobile/tablet/desktop breakpoints.
6. **Accessibility Notes** — Focus order, screen reader announcements, ARIA requirements.
7. **Navigation** — How this screen connects to others (entry/exit points).

Use ASCII/text diagrams for layout representation where helpful.
